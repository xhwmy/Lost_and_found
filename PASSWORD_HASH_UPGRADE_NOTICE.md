# 密码哈希算法统一升级说明

## 问题概述
我们发现系统中存在多种密码哈希算法混用的情况：
- `pbkdf2:sha256` (较旧的算法)
- `scrypt` (较新的算法)

这可能导致安全强度不一致的问题。

## 解决方案
我们已实现自动化的密码哈希算法升级机制：

### 1. 自动升级
- 当用户或管理员使用旧算法（pbkdf2:sha256）的账户登录时
- 系统会自动将密码哈希升级到新的scrypt算法
- 用户无需主动操作，升级过程透明

### 2. 代码实现
- 在 `user_views.py` 的登录函数中添加了算法检测和升级逻辑
- 在 `admin_views.py` 的登录函数中添加了算法检测和升级逻辑
- 创建了 `migrate_password_hashes.py` 脚本来处理算法检测和升级

### 3. 安全优势
- 所有账户最终都将使用更安全的scrypt算法
- 保持了向后兼容性，现有用户可以正常登录
- 逐步统一了系统的密码安全标准

## 影响
- 现有用户和管理员可以继续正常使用系统
- 首次使用旧算法登录的用户会看到"密码哈希算法已升级至更高安全级别"的通知
- 之后的所有登录都将受益于更高的安全算法

## 验证
您可以运行以下命令来验证算法升级是否正常工作：

```bash
python migrate_password_hashes.py
```

这将显示当前数据库中使用的算法分布情况。

---
此升级增强了系统的整体安全性，确保所有用户都受益于最新的密码哈希算法。
